STEP 03 DATA ANALYSIS

Data Analysis

데이터 분석 · PLAN, PREPROCESS & FIND INSIGHTS
이제 정리된 데이터로 분석을 수행할 단계입니다. 데이터 맵에 구성 된 데이터를 활용해, 먼저 분석 계획서를 작성하고, 분석 계획서를 바탕으로 분석작업을 수행해보시면 됩니다. 분석을 수행하다보면, 당연히 인사이트가 도출되지 않아, 분석의 방향을 바꿔야하는 경우가 생길 수 밖에 없습니다. 그렇다면, 분석 이후에 다시 분석계획서를 수정하고, 다른 기법으로 분석을 수행해야겠죠? 복잡하지만 이런 절차를 계속 수행하는 이유는, 현장에 많은 사람들이 이 프로젝트에 참여하거나 모니터링 하고 있기 때문입니다. 아래의 프로세스를 보고, 데이터 분석을 수행해 봅시다.

변수정의서까지 작성이 다 되면, 분석계획서도 작성을 진행해 주세요. 분석을 수행하기 전, 어떻게 이 데이터로 분석 작업을 수행할 지, 미리 계획을 작성 해야 합니다. 물론 현장에서는, 바로 데이터를 분석하는 경우도 있지만, 보통 팀끼리 협업을 하거나, 데이터를 분석하는 과정에서는 이 “데이터 분석 계획서”가 반드시 있어야만, 프로젝트를 원활히 진행할 수 있습니다.

그리고 데이터를 이용해, 분석에 용이하게 여러 전처리 작업을 수행해주세요. 특히 현장에서 발생한 진짜 데이터는 전처리 과정 없이 분석을 하게 되면 쉽게 인사이트를 찾기 어렵습니다. ‘층별화’를 통해, 다양한 범주 별로 데이터를 입체적으로 나누어 보셔야 합니다. 먼저 데이터의 구조와 타입을 빠르게 확인 후, 처리가 필요한 부분은 전처리를 해 나가며, 차근차근 인사이트를 얻어가는 작업을 해봅시다. 물론 ‘목표 변수’가 명확하게 설정 되어야겠지요?

또 분석 작업을 수행할 때, 유의 해야 할 점은 분석의 기법이 프로젝트 주제와 논리에 적합해야 한다는 것입니다. 간혹 데이터 머신러닝 기법이나 모델링 기법에 치우쳐서 프로젝트 주제에 맞게 인사이트를 도출할 수 있는 시각화 및 가설검정 기법들을 건너뛰곤 합니다. 앞서 프로젝트 배경 및 현황을 구성했던 것 처럼, 우리가 현장에서는 논리를 통해 분석이 수행되어야 하고, 분석에도 앞서 구성된 논리에 맞는 결과들이 나와야합니다. 모델링 기법을 시도해보는 것도 좋겠지만, 현장에서의 우선순위는 프로젝트 논리를 바탕으로 인사이트를 도출하는 것에 있다는 것을 절대 잊지 마세요!

분석 계획서 작성 → 전처리 → 분석 수행의 과정을 반복하며,
프로젝트 논리에 맞는 인사이트를 차근차근 도출해 나가는 단계입니다.
모델링에 치우치지 말고, 프로젝트 주제에 맞는 분석을
우선순위에 두고 진행해 주세요!
